# PeachyPet i18n 国际化设计文档

## 目标

在 PeachyPet 的功能型 UI 页面中引入中英文双语支持，用户可在设置页的新 Appearance section 中实时切换语言，无需重启 App。品牌/mascot 类页面保持英文原样不变。

## 不在范围内

- OnboardingView、PeachyDashboardView、MascotDetailView 不做翻译
- 不支持系统语言自动跟随（语言选择独立于系统 Locale）
- 不支持第三方翻译服务或远端字符串拉取
- 本次只做英文和中文，架构预留加语言的扩展点

## 架构

### LanguageManager

`@Observable final class LanguageManager`，全局单例 `LanguageManager.shared`。

**属性：**
- `language: AppLanguage`（枚举，初始值从 `UserDefaults("appLanguage")` 读取，默认 `.en`）
- `bundle: Bundle`（当前语言对应的 `.lproj` Bundle）

**切换逻辑（`setLanguage(_ lang: AppLanguage)`）：**
1. 在 `Bundle.main` 内找到 `<lang-code>.lproj` 目录路径
2. 用 `Bundle(path:)` 创建覆盖 bundle；若路径不存在则回退到 `Bundle.main`
3. 更新 `self.language` 和 `self.bundle`
4. 写 `UserDefaults("appLanguage")`

**全局函数：**
```swift
func t(_ key: String) -> String {
    LanguageManager.shared.bundle.localizedString(forKey: key, value: key, table: nil)
}
```

`value: key` 作为 fallback：key 找不到时直接显示 key 本身，不崩溃也不显示乱码。

### AppLanguage 枚举

```swift
enum AppLanguage: String, CaseIterable {
    case en = "en"
    case zh = "zh"
    var displayName: String {
        switch self {
        case .en: return "English"
        case .zh: return "中文"
        }
    }
}
```

## 文件结构

```
Sources/
  Utilities/
    LanguageManager.swift          ← @Observable 单例 + t() + AppLanguage
  Resources/
    en.lproj/
      Localizable.strings          ← 英文字符串（key = 语义 key）
    zh.lproj/
      Localizable.strings          ← 中文翻译
```

**Package.swift 改动：**  
resources 从 `.copy("Resources/...")` 分离 lproj 目录，改为 `.process("Resources/en.lproj")` 和 `.process("Resources/zh.lproj")`，让 SPM 将 `.lproj` 目录正确打包进 bundle。

**Info.plist 改动：**  
新增 `CFBundleLocalizations` 数组，值为 `["en", "zh"]`；确认 `CFBundleDevelopmentRegion` 为 `en`。

## 字符串 key 规范

格式：`<screen>.<element>`，全小写 snake_case。

示例：
```
settings.title = "Settings"
settings.connection = "Connection"
settings.language = "Language"
settings.auto_hide = "Auto-hide when no sessions"
settings.hide_delay = "Hide delay (seconds)"
settings.toast_enabled = "Show \"Task Completed\" toast"
settings.toast_duration = "Toast duration (seconds)"
permission.title = "Permission Requested"
permission.allow = "Allow"
permission.deny = "Deny"
overlay.task_completed = "Task completed"
overlay.idle = "Idle"
overlay.running = "Running"
```

**原则：**
- key 不用英文原文本身作 key（避免 key 本身就是翻译，造成英文回退难看）
- 品牌名词（PeachyPet、Claude Code、Codex、Ghostty 等）不翻译，直接嵌入字符串

## 设置页 Appearance Section

新增在所有现有 section **之前**（最顶部）：

```
── Appearance ──────────────
  Language    [English ▾]
```

实现：
```swift
Section {
    HStack {
        Text(t("settings.language"))
        Spacer()
        Picker("", selection: $selectedLanguage) {
            ForEach(AppLanguage.allCases, id: \.self) { lang in
                Text(lang.displayName).tag(lang)
            }
        }
        .pickerStyle(.menu)
        .labelsHidden()
        .onChange(of: selectedLanguage) { _, lang in
            LanguageManager.shared.setLanguage(lang)
        }
    }
} header: {
    Text(t("settings.appearance")).font(Constants.heading(size: 13, weight: .semibold))
}
```

`selectedLanguage` 用 `@State` 初始化为 `LanguageManager.shared.language`，`onChange` 触发切换，SwiftUI 因 `@Observable` 自动重渲染。

## 技术约束

- `t()` 访问 `LanguageManager.shared.bundle`，这是一个 `@Observable` 属性访问，SwiftUI 会在视图 body 内自动追踪依赖。**所有调用 `t()` 的视图必须在 body 内直接调用**，不能在 `onAppear`/`task` 之类的闭包内仅调用而不在 body 内引用任何 `LanguageManager` 属性。
- 为确保追踪生效，在 `LanguageManager` 上暴露一个 `var refreshToken: UUID`，`setLanguage` 时更新它；视图在 body 顶部 `let _ = LanguageManager.shared.refreshToken` 强制建立依赖（备用手段，若 `bundle` 属性追踪已足够则不需要）。
- 插值字符串示例：`Text(t("overlay.idle_since").replacingOccurrences(of: "{time}", with: timeString))`，避免 `String(format:)` 的 format specifier 本地化问题。

## 测试策略

- 单元测试：`LanguageManager.setLanguage(.zh)` 后 `t("settings.title")` 返回 `"设置"`；切换到 `.en` 后返回 `"Settings"`；未知 key 返回 key 本身不崩溃
- UI 验收：切换语言后 SettingsView、PermissionPromptView、SessionSwitcherView 立即显示对应语言，无需重启
- 回归：品牌名词不翻译（PeachyPet、Claude Code、Codex 等在两种语言下一致）
- 边界：`en.lproj` 缺失某 key 时，英文回退到 key 自身（不崩溃）
